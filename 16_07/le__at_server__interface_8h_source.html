   10 /**


   11  * @page c_atServer AT commands server


   12  *


   13  * @ref le_atServer_interface.h "API Reference"


   14  *


   15  * <HR>


   16  *


   17  * The AT commands server handles AT commands subscription on a requested serial device. The server


   18  * is complient with 3GPP 27.007, paragraph 4.0, 4.1 and 4.2.


   19  *


   20  * @section binding AT commands server and device binding


   21  *


   22  * le_atServer_Start() API must be called to bind a specific device with the AT commands server.


   23  * Multiple devices can be bound.


   24  * A device can be unbinded using le_atServer_Stop() API.


   25  *


   26  * @section subscription AT command subscription


   27  *


   28  * A new AT command can be added into the parser using le_atServer_Create(), and it can be deleted


   29  * thanks to le_atServer_Delete().


   30  * An AT command can be available only on one dedicated device, or on all binded AT devices, using


   31  * le_atServer_SetDevice() API.


   32  *


   33  * @section handler AT command handler


   34  *


   35  * To handle the AT command, the application has to subscribe a handler using


   36  * le_atServer_AddCommandHandler() (it may be removed with le_atServer_RemoveCommandHandler()).


   37  * The called handler (le_atServer_CommandHandlerRef_t prototype) can use


   38  * le_atServer_GetCommandName() to retrieve the received AT command string. It can also call


   39  * le_atServer_GetParameter() to retrieve a parameter of the AT command thanks to its index.


   40  *


   41  * @section responses AT command responses


   42  *


   43  * The application has the possibility to send intermediate responses thanks to


   44  * le_atServer_SendIntermediateResponse().


   45  * In all cases, the application must send a final response thanks to


   46  * le_atServer_SendFinalResponse(). The AT device is locked until the final response is sent. If no


   47  * answer is sent, the AT device will not accept any new AT commands (an error is returned in this


   48  * case).


   49  *


   50  * The application has also the possibility to send unsolicited responses to warn a host


   51  * application with le_atServer_SendUnsolicitedResponse() API.


   52  * This response is sent when no AT command is processing on the device (i.e. unsolicited


   53  * response is sent between the latest final response and the next reception of an AT command).


   54  * If an unsolicited response is sent whereas an AT command is in progress, the unsolicited response


   55  * is buffered and sent as soon as the AT device becomes available (i.e. the processing AT command


   56  * sends its final response).


   57  *


   58  * <HR>


   59  *


   60  * Copyright (C) Sierra Wireless Inc. Use of this work is subject to license.


   61  */
