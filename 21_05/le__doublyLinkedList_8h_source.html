    4Â  * @subpage le_doublyLinkedList.h "API Reference"
    5Â  *
    6Â  * <HR>
    7Â  *
    8Â  * A doubly linked list is a data structure consisting of a group of nodes linked together linearly.
    9Â  * Each node consists of data elements with links to the next node and previous nodes.  The main
   10Â  * advantage of linked lists (over simple arrays) is the nodes can be inserted and removed
   11Â  * anywhere in the list without reallocating the entire array. Linked list nodes don't
   12Â  * need to be stored contiguously in memory, but nodes then you can't access by
   13Â  * index, you have to be access by traversing the list.
   14Â  *
   15Â  * @section dls_createList Creating and Initializing Lists
   16Â  *
   17Â  * To create and initialize a linked list the user must create a le_dls_List_t typed list and assign
   18Â  * LE_DLS_LIST_INIT to it.  The assignment of LE_DLS_LIST_INIT can be done either when the list is
   19Â  * declared or after its declared.  The list <b>must</b> be initialized before it can be
   20Â  * used.
   21Â  *
   22Â  * @code
   23Â  * // Create and initialized the list in the declaration.
   24Â  * le_dls_List_t MyList = LE_DLS_LIST_INIT;
   25Â  * @endcode
   26Â  *
   27Â  * Or
   28Â  *
   29Â  * @code
   30Â  * // Create list.
   31Â  * le_dls_List_t MyList;
   32Â  *
   33Â  * // Initialize the list.
   34Â  * MyList = LE_DLS_LIST_INIT;
   35Â  * @endcode
   36Â  *
   37Â  * <b> Elements of le_dls_List_t MUST NOT be accessed directly by the user. </b>
   38Â  *
   39Â  *
   40Â  * @section dls_createNode Creating and Accessing Nodes
   41Â  *
   42Â  * Nodes can contain any data in any format and is defined and created by the user.  The only
   43Â  * requirement for nodes is that it must contain a @c le_dls_Link_t link member. The link member must
   44Â  * be initialized by assigning LE_DLS_LINK_INIT to it before it can be used.  Nodes can then be
   45Â  * added to the list by passing their links to the add functions (le_dls_Stack(), le_dls_Queue(),
   46Â  * etc.).  For example:
   47Â  *
   48Â  * @code
   49Â  * // The node may be defined like this.
   50Â  * typedef struct
   51Â  * {
   52Â  *      dataType someUserData;
   53Â  *      ...
   54Â  *      le_dls_Link_t myLink;
   55Â  *
   56Â  * }
   57Â  * MyNodeClass_t;
   58Â  *
   59Â  * // Create and initialize the list.
   60Â  * le_dls_List_t MyList = LE_DLS_LIST_INIT;
   61Â  *
   62Â  * void foo (void)
   63Â  * {
   64Â  *     // Create the node.  Get the memory from a memory pool previously created.
   65Â  *     MyNodeClass_t* myNodePtr = le_mem_ForceAlloc(MyNodePool);
   66Â  *
   67Â  *     // Initialize the node's link.
   68Â  *     myNodePtr->myLink = LE_DLS_LINK_INIT;
   69Â  *
   70Â  *     // Add the node to the head of the list by passing in the node's link.
   71Â  *     le_dls_Stack(&MyList, &(myNodePtr->myLink));
   72Â  * }
   73Â  * @endcode
   74Â  *
   75Â  * The links in the nodes are actually added to the list and not the nodes themselves. This
   76Â  * allows a node to be included on multiple lists through links
   77Â  * added to different lists. It also allows linking different type nodes in a list.
   78Â  *
   79Â  * To obtain the node itself, use the @c CONTAINER_OF macro defined in
   80Â  * le_basics.h.  Here's a code sample using CONTAINER_OF to obtain the node:
   81Â  *
   82Â  * @code
   83Â  * // Assuming mylist has been created and initialized and is not empty.
   84Â  * le_dls_Link_t* linkPtr = le_dls_Peek(&MyList);
   85Â  *
   86Â  * // Now we have the link but still need the node to access user data.
   87Â  * // We use CONTAINER_OF to get a pointer to the node given the node's link.
   88Â  * if (linkPtr != NULL)
   89Â  * {
   90Â  *     MyNodeClass_t* myNodePtr = CONTAINER_OF(linkPtr, MyNodeClass_t, myLink);
   91Â  * }
   92Â  * @endcode
   93Â  *
   94Â  * The user is responsible for creating and freeing memory for all nodes; the linked list module
   95Â  * only manages the links in the nodes.  The node must be removed from all lists before its
   96Â  * memory can be freed.
   97Â  *
   98Â  * <b>The elements of @c le_dls_Link_t MUST NOT be accessed directly by the user.</b>
   99Â  *
  100Â  *
  101Â  * @section dls_add Adding Links to a List
  102Â  *
  103Â  * To add nodes to a list, pass the node's link to one of these functions:
  104Â  *
  105Â  * - @c le_dls_Stack() - Adds the link to the head of the list.
  106Â  * - @c le_dls_Queue() - Adds the link to the tail of the list.
  107Â  * - @c le_dls_AddAfter() - Adds the link to a list after another specified link.
  108Â  * - @c le_dls_AddBefore() - Adds the link to a list before another specified link.
  109Â  *
  110Â  *
  111Â  * @section dls_remove Removing Links from a List
  112Â  *
  113Â  * To remove nodes from a list, use one of these functions:
  114Â  *
  115Â  * - @c le_dls_Pop() - Removes and returns the link at the head of the list.
  116Â  * - @c le_dls_PopTail() - Removes and returns the link at the tail of the list.
  117Â  * - @c le_dls_Remove() - Remove a specified link from the list.
  118Â  *
  119Â  *
  120Â  * @section dls_peek Accessing Links in a List
  121Â  *
  122Â  * To access a link in a list without removing the link, use one of these functions:
  123Â  *
  124Â  * - @c le_dls_Peek() - Returns the link at the head of the list without removing it.
  125Â  * - @c le_dls_PeekTail() - Returns the link at the tail of the list without removing it.
  126Â  * - @c le_dls_PeekNext() - Returns the link next to a specified link without removing it.
  127Â  * - @c le_dls_PeekPrev() - Returns the link previous to a specified link without removing it.
  128Â  *
  129Â  *
  130Â  * @section dls_swap Swapping Links
  131Â  *
  132Â  * To swap two links, use:
  133Â  *
  134Â  * - @c le_dls_Swap() - Swaps the position of two links in a list.
  135Â  *
  136Â  * @section dls_sort Sorting Lists
  137Â  *
  138Â  * To sort a list use:
  139Â  *
  140Â  * @c le_dls_Sort() - Sorts a list
  141Â  *
  142Â  * @section dls_query Querying List Status
  143Â  *
  144Â  * These functions can be used to query a list's current status:
  145Â  *
  146Â  * - @c le_dls_IsEmpty() - Checks if a given list is empty.
  147Â  * - @c le_dls_IsInList() - Checks if a specified link is in the list.
  148Â  * - @c le_dls_IsHead() - Checks if a specified link is at the head of the list.
  149Â  * - @c le_dls_IsTail() - Checks if a specified link is at the tail of the list.
  150Â  * - @c le_dls_NumLinks() - Checks the number of links currently in the list.
  151Â  * - @c le_dls_IsListCorrupted() - Checks if the list is corrupted.
  152Â  *
  153Â  *
  154Â  * @section dls_fifo Queues and Stacks
  155Â  *
  156Â  * This implementation of linked lists can be used for either queues or stacks.
  157Â  *
  158Â  * To use the list as a queue, restrict additions to the list to @c le_dls_Queue() and removals
  159Â  * from the list to @c le_dls_Pop().
  160Â  *
  161Â  * To use the list as a stack,  restrict additions to the list to @c le_dls_Stack() and removals
  162Â  * from the list to @c le_dls_Pop().
  163Â  *
  164Â  *
  165Â  * @section dls_synch Thread Safety and Re-Entrancy
  166Â  *
  167Â  * All linked list function calls are re-entrant and thread safe themselves, but if the nodes and/or
  168Â  * list object are shared by multiple threads, explicit steps must be taken to maintain mutual

  170Â  * @ref c_mutex "mutex" or some other form of thread synchronization to ensure only one thread
  171Â  * accesses the list at a time.
