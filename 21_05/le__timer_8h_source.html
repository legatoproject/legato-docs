    7Â  *
    8Â  * This module provides an API for managing and using timers.
    9Â  *
   10Â  * @note
   11Â  * This is an initial version of the API that only provides support for relative timers (e.g., expires
   12Â  * in 10 seconds).  Absolute timers allow a specific time/date to be used, and will be supported in a
   13Â  * future version of this API.
   14Â  *
   15Â  *
   16Â  * @section timer_objects Creating/Deleting Timer Objects
   17Â  *
   18Â  * Timers are created using le_timer_Create(). The timer name is used for diagnostic purposes only.
   19Â  *
   20Â  * The following attributes of the timer can be set:
   21Â  *  - le_timer_SetHandler()
   22Â  *  - le_timer_SetInterval() (or le_timer_SetMsInterval())
   23Â  *  - le_timer_SetRepeat()
   24Â  *  - le_timer_SetContextPtr()
   25Â  *
   26Â  * The following attributes of the timer can be retrieved:
   27Â  *  - le_timer_GetInterval() (or le_timer_GetMsInterval())
   28Â  *  - le_timer_GetContextPtr()
   29Â  *
   30Â  * The repeat count defaults to 1, so that the timer is initially a one-shot timer. All the other
   31Â  * attributes must be explicitly set.  At a minimum, the interval must be set before the timer can be
   32Â  * used.  Note that these attributes can only be set if the timer is not currently running; otherwise,
   33Â  * an error will be returned.
   34Â  *
   35Â  * Timers must be explicitly deleted using le_timer_Delete(). If the timer is currently running,
   36Â  * it'll be stopped before being deleted. If a timer uses le_timer_SetContextPtr(), and the
   37Â  * context pointer is allocated memory, then the context pointer must be freed when deleting the timer.
   38Â  * The following function can be used for this:
   39Â  * @code
   40Â  * static void DeleteTimerAndFreePtr(le_timer_Ref_t t)
   41Â  * {
   42Â  *     le_timer_Stop( t );
   43Â  *     free( le_timer_GetContextPtr( t ) );
   44Â  *     le_timer_Delete( t );  // timer ref is now invalid
   45Â  * }
   46Â  * @endcode
   47Â  * You can call this function anywhere, including in the timer handler.
   48Â  *
   49Â  * @section timer_usage Using Timers
   50Â  *
   51Â  * A timer is started using le_timer_Start(). If it's already running, then it won't be
   52Â  * modified; instead an error will be returned. To restart a currently running timer, use
   53Â  * le_timer_Restart().
   54Â  *
   55Â  * A timer is stopped using le_timer_Stop().  If it's not currently running, an error
   56Â  * will be returned, and nothing more will be done.
   57Â  *
   58Â  * To determine if the timer is currently running, use le_timer_IsRunning().
   59Â  *
   60Â  * To find out how much time is remaining before the next expiry, call either
   61Â  * le_timer_GetTimeRemaining() or le_timer_GetMsTimeRemaining().
   62Â  *
   63Â  * When a timer expires, if the timer expiry handler is set by le_timer_SetHandler(), the
   64Â  * handler will be called with a reference to the expired timer. If additional data is required in the
   65Â  * handler, le_timer_SetContextPtr() can be used to set the appropriate context before starting the
   66Â  * timer, and le_timer_GetContextPtr() can be used to retrieve the context while in the handler.
   67Â  * In addition, a suspended system will also wake up by default if the timer expires. If this behaviour
   68Â  * is not desired, user can disable the wake up by passing false into le_timer_SetWakeup().
   69Â  *
   70Â  * The number of times that a timer has expired can be retrieved by le_timer_GetExpiryCount(). This
   71Â  * count is independent of whether there is an expiry handler for the timer.
   72Â  *
   73Â  * @section le_timer_thread Thread Support
   74Â  *
   75Â  * A timer should only be used by the thread that created it. It's not safe for a thread to use
   76Â  * or manipulate a timer that belongs to another thread. The timer expiry handler is called by the
   77Â  * event loop of the thread that starts the timer.
   78Â  *
   79Â  * The call to the timer expiry handler may not occur immediately after the timer expires, depending
   80Â  * on which other functions are called from the event loop. The amount of delay is entirely
   81Â  * dependent on other work done in the event loop. For a repeating timer, if this delay is longer
   82Â  * than the timer period, one or more timer expiries may be dropped. To reduce the likelihood of
   83Â  * dropped expiries, the combined execution time of all handlers called from the event loop should
   84Â  *  ideally be less than the timer period.
   85Â  *
   86Â  * See @ref c_eventLoop for details on running the event loop of a thread.
   87Â  *
   88Â  * @section le_timer_suspend Suspend Support
   89Â  *
   90Â  * The timer runs even when system is suspended. <br>
   91Â  * If the timer expires while the system is suspended, it will wake up the system.
   92Â  *
   93Â  * @section timer_errors Fatal Errors
   94Â  *
   95Â  * The process will exit under any of the following conditions:
   96Â  *  - If an invalid timer object is given to:
