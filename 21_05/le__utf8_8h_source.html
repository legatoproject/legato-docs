    2Â  * @page c_utf8 UTF-8 String Handling API
    3Â  *
    4Â  * @subpage le_utf8.h "API Reference"
    5Â  *
    6Â  * <HR>
    7Â  *
    8Â  * This module implements safe and easy to use string handling functions for null-terminated strings
    9Â  * with UTF-8 encoding.
   10Â  *
   11Â  * UTF-8 is a variable length character encoding that supports every character in the Unicode
   12Â  * character set. UTF-8 has become the dominant character encoding because it is self synchronizing,
   13Â  * compatible with ASCII, and avoids the endian issues that other encodings face.
   14Â  *
   15Â  *  @section utf8_encoding UTF-8 Encoding
   16Â  *
   17Â  * UTF-8 uses between one and four bytes to encode a character as illustrated in the following
   18Â  * table.
   19Â  *
   20Â  * <table>
   21Â  * <tr> <th> Byte 1   </th> <th> Byte 2   </th> <th> Byte 3   </th> <th> Byte 4   </th> </tr>
   22Â  * <tr> <td> 0xxxxxxx </td> <td>          </td> <td>          </td> <td>          </td> </tr>
   23Â  * <tr> <td> 110xxxxx </td> <td> 10xxxxxx </td> <td>          </td> <td>          </td> </tr>
   24Â  * <tr> <td> 1110xxxx </td> <td> 10xxxxxx </td> <td> 10xxxxxx </td> <td>          </td> </tr>
   25Â  * <tr> <td> 11110xxx </td> <td> 10xxxxxx </td> <td> 10xxxxxx </td> <td> 10xxxxxx </td> </tr>
   26Â  * </table>
   27Â  *
   28Â  * Single byte codes are used only for the ASCII values 0 through 127.  In this case, UTF-8 has the
   29Â  * same binary value as ASCII, making ASCII text valid UTF-8 encoded Unicode.  All ASCII strings are
   30Â  * UTF-8 compatible.
   31Â  *
   32Â  * Character codes larger than 127 have a multi-byte encoding consisting of a leading byte and one
   33Â  * or more continuation bytes.
   34Â  *
   35Â  * The leading byte has two or more high-order 1's followed by a 0 that can be used to determine the
   36Â  * number bytes in the character without examining the continuation bytes.
   37Â  *
   38Â  * The continuation bytes have '10' in the high-order position.
   39Â  *
   40Â  * Single bytes, leading bytes and continuation bytes can't have the same values. This means that
   41Â  * UTF-8 strings are self-synchronized, allowing the start of a character to be found by backing up
   42Â  * at most three bytes.
   43Â  *
   44Â  * @c le_utf8_EncodeUnicodeCodePoint() provides a function that is able to encode any unicode code
   45Â  * point into a sequence of bytes that represents the utf-8 encoding of the codepoint.  The function
   46Â  * @c le_utf8_DecodeUnicodeCodePoint() implements the inverse function.  It converts a UTF-8 encoded
   47Â  * character into the corresponding unicode code point.
   48Â  *
   49Â  *  @section utf8_copy Copy and Append
   50Â  *
   51Â  * @c le_utf8_Copy() copies a string to a specified buffer location.
   52Â  *
   53Â  * @c le_utf8_Append() appends a string to the end of another string by copying the source string to
   54Â  * the destination string's buffer starting at the null-terminator of the destination string.
   55Â  *
   56Â  * The @c le_uft8_CopyUpToSubStr() function is like le_utf8_Copy() except it copies only up to, but
   57Â  * not including, a specified string.
   58Â  *
   59Â  *  @section utf8_trunc Truncation
   60Â  *
   61Â  * Because UTF-8 is a variable length encoding, the number of characters in a string is not
   62Â  * necessarily the same as the number bytes in the string.  When using functions like le_utf8_Copy()
   63Â  * and le_utf8_Append(), the size of the destination buffer, in bytes, must be provided to avoid
   64Â  * buffer overruns.
   65Â  *
   66Â  * The copied string is truncated because of limited space in the destination buffer, and the
   67Â  * destination buffer may not be completely filled.  This can occur during the copy process if the
   68Â  * last character to copy is more than one byte long and will not fit within the buffer.
   69Â  *
   70Â  * The character is not copied and a null-terminator is added.  Even though we have not filled the
   71Â  * destination buffer, we have truncated the copied string.  Essentially, functions like
   72Â  * le_utf8_Copy() and le_utf8_Append() only copy complete characters, not partial characters.
   73Â  *
   74Â  * For le_utf8_Copy(), the number of bytes actually copied is returned in the numBytesPtr parameter.
   75Â  * This parameter can be set to NULL if the number of bytes copied is not needed.  le_utf8_Append()
   76Â  * and le_uft8_CopyUpToSubStr() work similarly.
   77Â  *
   78Â  * @code
   79Â  * // In this code sample, we need the number of bytes actually copied:
   80Â  * size_t numBytes;
   81Â  *
   82Â  * if (le_utf8_Copy(destStr, srcStr, sizeof(destStr), &numBytes) == LE_OVERFLOW)
   83Â  * {
   84Â  *     LE_WARN("'%s' was truncated when copied.  Only %d bytes were copied.", srcStr, numBytes);
   85Â  * }
   86Â  *
   87Â  * // In this code sample, we don't care about the number of bytes copied:
   88Â  * LE_ASSERT(le_utf8_Copy(destStr, srcStr, sizeof(destStr), NULL) != LE_OVERFLOW);
   89Â  * @endcode
   90Â  *
   91Â  *  @section utf8_stringLength String Lengths
   92Â  *
   93Â  * String length may mean either the number of characters in the string or the number of bytes in
   94Â  * the string.  These two meanings are often used interchangeably because in ASCII-only encodings
   95Â  * the number of characters in a string is equal to the number of bytes in a string. But this is not
   96Â  * necessarily true with variable length encodings such as UTF-8. Legato provides both a
   97Â  * le_utf8_NumChars() function and a le_utf8_NumBytes() function.
   98Â  *
   99Â  * @c le_utf8_NumBytes() must be used when determining the memory size of a string.
  100Â  * @c le_utf8_NumChars() is useful for counting the number of characters in a string (ie. for
  101Â  * display purposes).
  102Â  *
  103Â  *  @section utf8_charLength Character Lengths
  104Â  *
  105Â  * The function le_utf8_NumBytesInChar() can be used to determine the number of bytes in a character
  106Â  * by looking at its first byte.  This is handy when reading a UTF-8 string from an input stream.
  107Â  * When the first byte is read, it can be passed to le_utf8_NumBytesInChar() to determine how many
  108Â  * more bytes need to be read to get the rest of the character.
  109Â  *
  110Â  *  @section utf8_format Checking UTF-8 Format
  111Â  *
  112Â  * As can be seen in the @ref utf8_encoding section, UTF-8 strings have a specific byte sequence.
  113Â  * The @c le_utf8_IsFormatCorrect() function can be used to check if a string conforms to UTF-8
  114Â  * encoding.  Not all valid UTF-8 characters are valid for a given character set;
  115Â  * le_utf8_IsFormatCorrect() does not check for this.
  116Â  *
  117Â  *  @section utf8_parsing String Parsing
  118Â  *
  119Â  * To assist with converting integer values from UTF-8 strings to binary numerical values,
  120Â  * le_utf8_ParseInt() is provided.
  121Â  *
  122Â  * More parsing functions may be added as required in the future.
  123Â  *
  124Â  * <hr>
  125Â  *
