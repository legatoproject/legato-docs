    2Â  * @page c_flock File Locking API

    4Â  * @subpage le_fileLock.h "API Reference"
    5Â  *
    6Â  * <HR>
    7Â  *
    8Â  * File locking is a form of IPC used to synchronize multiple processes' access to
    9Â  * common files.
   10Â  *
   11Â  * This API provides a co-operative file locking mechanism that can be used by multiple processes
   12Â  * and/or threads to sychronize reads and writes to common files.
   13Â  *
   14Â  * This API only supports regular files. Attempts to use this API on sockets, devices, etc. results
   15Â  * in undefined behaviour.
   16Â  *
   17Â  * @section c_flock_cooperative Co-operative File Locking
   18Â  *
   19Â  * Co-operative file locks (also known as advisory file locks) means that the processes and threads
   20Â  * must co-operate to synchronize their access to the file.  If a process or thread simply ignores
   21Â  * the lock and accesses the file then access synchronization errors may occur.
   22Â  *
   23Â  * @section c_flock_locks Locking Files
   24Â  *
   25Â  * There are two types of locks that can be applied: read lock and write lock.  A file
   26Â  * can have multiple simultaneous read locks, but can only have one write lock. Also, a
   27Â  * file can only have one type of lock on it at one time. A file may be locked
   28Â  * for reading if the file is unlocked or if there are read locks on the file, but to lock a file
   29Â  * for writing the file must be unlocked.
   30Â  *
   31Â  * Use @c le_flock_Open() to lock a file and open it for access. When attempting to lock a file that
   32Â  * already has an incompatible lock on it, @c le_flock_Open() will block until it can obtain the lock.
   33Â  * Call @c le_flock_Close() to close the file and remove the lock on the file.
   34Â  *
   35Â  * This code sample shows four processes attempting to access the same file.  Assume
   36Â  * that all the calls to le_flock_Open() in the example occur in chronological order as they appear:
   37Â  *
   38Â  * @code
   39Â  *      // Code in Process 1.
   40Â  *
   41Â  *      // Lock the file for reading.
   42Â  *      int fd = le_flock_Open("foo", LE_FLOCK_READ);  // This call will not block.
   43Â  *
   44Â  *      // Read from the file.
   45Â  *      ...
   46Â  *
   47Â  *      // Close the file and release the lock.
   48Â  *      le_flock_Close(fd);
   49Â  * -------------------------------------------------------------------------------------------------
   50Â  *
   51Â  *      // Code in Process 2.
   52Â  *
   53Â  *      // Lock the file for reading.
   54Â  *      int fd = le_flock_Open("foo", LE_FLOCK_READ);  // This call will not block.
   55Â  *
   56Â  *      // Read from the file.
   57Â  *      ...
   58Â  *
   59Â  *      // Close the file and release the lock.
   60Â  *      le_flock_Close(fd);
   61Â  * -------------------------------------------------------------------------------------------------
   62Â  *
   63Â  *      // Code in Process 3.
   64Â  *
   65Â  *      // Lock the file for writing.
   66Â  *      int fd = le_flock_Open("foo", LE_FLOCK_WRITE);  // This call will block until both Process 1
   67Â  *                                                      // and Process 2 removes their locks.
   68Â  *
   69Â  *      // Write to the file.
   70Â  *      ...
   71Â  *
   72Â  *      // Close the file and release the lock.
   73Â  *      le_flock_Close(fd);
   74Â  * @endcode
   75Â  *
   76Â  * This sample shows that Process 2 obtains the read lock even though Process 1 already
   77Â  * has a read lock on the file. Process 3 is blocked because it's attempting a
   78Â  * write lock on the file.  Process 3 is blocked until both Process 1 and 2 remove their locks.
   79Â  *
   80Â  * When multiple processes are blocked waiting to obtain a lock on the file, it's unspecified which
   81Â  * process will obtain the lock when the file becomes available.
   82Â  *
   83Â  * The le_flock_Create() function can be used to create, lock and open a file in one function call.
   84Â  *
   85Â  * @section c_flock_streams Streams
   86Â  *
   87Â  * The functions @c le_flock_OpenStream() and @c le_flock_CreateStream() can be used to obtain a file
   88Â  * stream to a locked file.  @c le_flock_CloseStream() is used to close the stream and remove the lock.
   89Â  * These functions are analogous to le_flock_Open(), le_flock_Create() and le_flock_Close() except
   90Â  * that they return file streams rather than file descriptors.
   91Â  *
   92Â  * @section c_flock_nonblock Non-blocking
   93Â  *
   94Â  * Functions le_flock_Open(), le_flock_Create(), le_flock_OpenStream() and
   95Â  * le_flock_CreateStream() always block if there is an incompatible lock on the file.  Functions
   96Â  * le_flock_TryOpen(), le_flock_TryCreate(), le_flock_TryOpenStream() and
   97Â  * le_flock_TryCreateStream() are their non-blocking counterparts.
   98Â  *
   99Â  * @section c_flock_threads Multiple Threads
  100Â  *
  101Â  * All functions in this API are thread-safe; processes and threads can use this API to
  102Â  * synchronize their access to files.
  103Â  *
  104Â  * @section c_flock_replicateFd Replicating File Descriptors
  105Â  *
  106Â  * File locks are contained in the file descriptors that are returned by le_flock_Open() and
  107Â  * le_flock_Create() and in the underlying file descriptors of the file streams returned by
  108Â  * le_flock_OpenStream() and le_flock_CreateStream().
  109Â  *
  110Â  * File descriptors are closed the locks are automatically removed.  Functions
  111Â  * le_flock_Close() and le_flock_CloseStream() are provided as a convenience.  When a process dies,
  112Â  * all of its file descriptors are closed and any file locks they may contain are removed.
  113Â  *
  114Â  * If a file descriptor is replicated either through dup() or fork(), the file lock will also be
  115Â  * replicated in the new file descriptor:
  116Â  *
  117Â  * @code
  118Â  *      int oldfd = le_flock_Open("foo", LE_FLOCK_READ);  // Place a read lock on the file "foo".
  119Â  *      int newfd = dup(oldfd);
  120Â  *
  121Â  *      le_flock_Close(oldfd); // Closes the fd and removes the lock.
  122Â  * @endcode
  123Â  *
  124Â  * There must still be a read lock on the file "foo" because newfd
  125Â  * has not been closed.
  126Â  *
  127Â  * This behaviour can be used to pass file locks from a parent to a child through a fork() call.
  128Â  * The parent can obtain the file lock, fork() and close its file descriptor.  Now the child has
  129Â  * exclusive possession of the file lock.
  130Â  *
  131Â  * @section c_flock_limitations Limitations
  132Â  *
  133Â  * Here are some limitations to the file locking mechanisms in this API:
  134Â  *
  135Â  * The file locks in this API are advisory only, meaning that a process may
  136Â  * simply ignore the lock and access the file anyways.
  137Â  *
  138Â  * This API does not detect deadlocks and a process may deadlock itself.  For example:
  139Â  *
  140Â  * @code
  141Â  *      int fd1 = le_flock_Open("foo", LE_FLOCK_READ);   // Obtains a read lock on the file.
  142Â  *      int fd2 = le_flock_Open("foo", LE_FLOCK_WRITE);  // This call will block forever.
  143Â  * @endcode
  144Â  *
  145Â  * This API only permits whole files to be locked, not portions of a
  146Â  * file.
  147Â  *
  148Â  * Many NFS implementations don't recognize locks used by this API.
  149Â  *
  150Â  * <HR>
